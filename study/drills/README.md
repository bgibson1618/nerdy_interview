# Code-review drills

Two kinds of timed, planted-bug review reps for the Nerdy code-review interview.
Each drill plants a realistic mix of **Blocker / Should-fix / Nit** defects plus one
**good habit to praise** — review it out loud, severity-tag your findings, *then* check
the key.

## Static drills (snippet + answer key in one file)

Single-file PRs you review right here. The answer key is a `<details>` block — don't peek.

| # | Drill | Area | Blockers planted |
|---|-------|------|------------------|
| 01 | [TypeScript type-safety](01-ts-types.md) | TS soundness | unsound `as` cast on untrusted input, `!` crash, NaN invoices |
| 02 | [React hooks & state](02-react-hooks.md) | React | `dangerouslySetInnerHTML` XSS, stale-closure timer |
| 03 | [SQL / data access](03-sql.md) | MySQL | SQL injection, un-transactioned money move, null-deref |
| 04 | [GraphQL resolvers](04-graphql.md) | GraphQL | IDOR, JSON-scalar row leak, non-null propagation |
| 05 | [gRPC service](05-grpc.md) | gRPC | no auth metadata, not-found crash, OK-on-error status |
| 06 | [OAuth 2.0 flow](06-oauth.md) | OAuth | missing `state` (CSRF), unverified ID token, non-HttpOnly token |

Drills 03–06 cover the integration surface (and the GraphQL/gRPC areas worth extra prep);
their answer keys *teach* the concept, not just name it.

## PR review drills (review on GitHub)

Multi-file feature PRs opened against `main` — review them in the real GitHub PR UI
(Files-changed tab, inline comments), the way the interview will feel.

- Keys: **[pr-review-keys.md](pr-review-keys.md)** (spoilers — review the PR first).
- Open PRs: see the repo's **Pull requests** tab. The branches:
  - `feat/payment-webhook` — webhook receiver (signature, idempotency, reliability).
  - `feat/students-rest-api` — REST CRUD (mass assignment, IDOR, pagination, status codes).
  - `feat/jwt-auth-middleware` — JWT auth (decode-vs-verify, fail-open, token handling).

The keys live on `main` and are **not** in any PR diff, so reviewing a PR won't spoil it.
