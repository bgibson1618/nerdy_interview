# Code-review drills

Two kinds of timed, planted-bug review reps for the Nerdy code-review interview.
Each drill plants a realistic mix of **Blocker / Should-fix / Nit** defects plus one
**good habit to praise** — review it out loud, severity-tag your findings, *then* check
the key.

## Static drills (snippet + answer key in one file)

Single-file PRs you review right here. The answer key is a `<details>` block — don't peek.

| # | Drill | Area | Blockers planted |
|---|-------|------|------------------|
| 01 | [TypeScript type-safety](01-ts-types.md) | TS soundness | unsound `as` cast on untrusted input, `!` crash, NaN invoices |
| 02 | [React hooks & state](02-react-hooks.md) | React | `dangerouslySetInnerHTML` XSS, stale-closure timer |
| 03 | [SQL / data access](03-sql.md) | MySQL | SQL injection, un-transactioned money move, null-deref |
| 04 | [GraphQL resolvers](04-graphql.md) | GraphQL | IDOR, JSON-scalar row leak, non-null propagation |
| 05 | [gRPC service](05-grpc.md) | gRPC | no auth metadata, not-found crash, OK-on-error status |
| 06 | [OAuth 2.0 flow](06-oauth.md) | OAuth | missing `state` (CSRF), unverified ID token, non-HttpOnly token |
| 07 | [Async / concurrency](07-async-concurrency.md) | JS/TS async | TOCTOU double-grant, lost-rejection, `forEach(async)` |
| 08 | [Error handling & resilience](08-error-handling-resilience.md) | resilience | retry double-charge, no timeout, client error leak |
| 09 | [Test quality](09-testing-quality.md) | testing | assertion-free + false-confidence tests on a critical path |
| 10 | [Web security grab-bag](10-security-grabbag.md) | security | XSS, SSRF, path traversal, CSRF, open redirect |
| 11 | [REST API design](11-rest-api-design.md) | REST | IDOR, state-changing GET (CSRF), mass assignment |
| 12 | [Node.js backend gotchas](12-node-backend-gotchas.md) | Node | NoSQL injection, stream crash, secret in logs, sync `pbkdf2` |
| 13 | [Caching & performance](13-perf-caching.md) | caching | cross-user cache-key leak, no invalidation, stampede |
| 14 | [SQL Server / T-SQL](14-sqlserver-tsql.md) | SQL Server | dynamic-SQL injection, `NOLOCK` dirty read, no transaction |
| 15 | [Advanced TypeScript](15-typescript-advanced.md) | TS soundness | `as unknown as` trust-boundary cast, lost exhaustiveness |
| 16 | [React perf & a11y](16-react-perf-a11y.md) | React | div-as-button keyboard lockout, `key={index}` grade swap |

Drills 03–06 and 10–16 cover the integration/security/perf surface (and the GraphQL/gRPC areas
worth extra prep); their answer keys *teach* the concept, not just name it. **A few answer keys
are deliberately not exhaustive** — a sharp reviewer can find issues the key doesn't list, which
is itself the skill being drilled. (07–16 were cross-backend QA'd; the confirmed key errors were
corrected — see `EXPERIMENTS.md` #9.)

## PR review drills (review on GitHub)

Multi-file feature PRs opened against `main` — review them in the real GitHub PR UI
(Files-changed tab, inline comments), the way the interview will feel.

- Keys: **[pr-review-keys.md](pr-review-keys.md)** (spoilers — review the PR first).
- Open PRs: see the repo's **Pull requests** tab. The branches:
  - `feat/payment-webhook` (#1) — webhook receiver (signature, idempotency, reliability).
  - `feat/students-rest-api` (#2) — REST CRUD (mass assignment, IDOR, pagination, status codes).
  - `feat/jwt-auth-middleware` (#3) — JWT auth (decode-vs-verify, fail-open, token handling).
  - `feat/password-reset` (#4) — reset flow (predictable token, no expiry, enumeration).
  - `feat/avatar-upload` (#5) — file upload (path traversal, content-type XSS, no auth).
  - `feat/graphql-notes` (#6) — GraphQL resolvers (IDOR, unauth delete, N+1, row leak).
  - `feat/course-cache` (#7) — caching (cross-user key leak, no invalidation, stampede).
  - `feat/email-queue-worker` (#8) — queue consumer (ack-before-process, poison retry).

The keys live on `main` and are **not** in any PR diff, so reviewing a PR won't spoil it.
